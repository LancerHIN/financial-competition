# A 榜错题归因分析（answer-final-verified.csv，78 分 / 实测 73 题精确匹配 gold）

> 数据源：`answer-final-verified.csv`（最终提交）、`logs/analysis_report_with_gold.csv`（gold 列 +
> pipeline 原始答案）、`evidence.json`（每题 judgements + 证据归属）。
> 逐题明细见 `logs/_wrong_detail.txt`。
>
> 注：按"精确匹配 gold"统计有 27 题不一致；官方 78 分说明多选题给了部分分，
> 但**精确匹配 = 满分**才是优化目标，所以下面按 27 题精确错题分析。

**最大杠杆 = 多选过包含（13 题，48%）。** 这一类几乎全部是同一个机制（见第三节）。

## 二、致命教训：人工改答案是"抛硬币"，净 +7 但引入 5 个倒退

对比 pipeline 原始答案 vs 人工核对后的 verified：

| 人工动作 | 题数 | 题号 |
|---|---|---|
| 改对了（wrong→right） | 12 | fc_a_003/011/017/020, fin_a_004/005/012/014, ins_a_003, res_a_007/017/019 |
| **改错了（right→wrong）** | **5** | **fc_a_013, fc_a_019, reg_a_001, res_a_018, res_a_020** |
| 改了还是错（wrong→wrong） | 5 | fc_a_001, fin_a_008, fin_a_020, ins_a_019, res_a_002 |

**5 个 pipeline 本来做对、被人工手动改错的题，是纯送分倒退：**
- fc_a_013：pipeline=B（对）→ 人工改成 A（错）
- fc_a_019：pipeline=B（对）→ 人工加成 AB（错）
- reg_a_001：pipeline=AC（对）→ 人工加成 ACD（错）
- res_a_018：pipeline=A（对）→ 人工翻成 B（错）
- res_a_020：pipeline=A（对）→ 人工加成 AD（错）

**规律**：人工改动里"加选项"和"翻转 tf"风险最高。人工倾向于"宁可多选/翻面"，
正好踩中多选过包含这个系统性陷阱。**结论：未来禁止凭直觉手动加选项，
任何改动必须有逐条原文佐证 + 端到端复跑，否则不如信任 pipeline。**

## 三、根因 1（核心）：多选过包含 —— judge 对"看似匹配"的干扰项判 supported

13 道过包含题里，被错误多选的选项**全部**被 judge 打了 `supported c=1.0`。
拆开看，干扰项有四种"看似匹配实则不符"的套路，judge 都没识破：

### 套路 A：主语/主体张冠李戴（跨文档串味）— **这就是"证据归属准确度"问题**
- **res_a_002 / res_a_004 / res_a_012**：选项把 A 文档的指标安到 B 主体上，或把研报里
  "博通/Lumentum/台湾地区"的预测说成"芯原股份/韩国"。judge 看到数值匹配就判 supported，
  没核对**陈述主语 = 证据主语**。
  - res_a_004 选项 C："韩国银保渠道...支撑了韩国**寿险管理体系**"——证据只说"支撑保费增长"，
    judge 自己都写了"即...的重要部分"在**脑补等价**。
  - res_a_002 选项 B：judge 原话"虽然原文用词为...结合语境...**通常研报中此类表述即对应选项含义**"
    —— 这是教科书级的"自我合理化幻觉"。
- **fc_a_019**：选项 D 主语错（安克创新属第一份文档非第二份），judge 推理里已经发现
  "证据 E9/E24 属 text04...text07 中并未出现安克创新"，**但最终仍判 supported**——
  推理正确、结论却反向，说明 status 字段与 reason 脱节。

### 套路 B：限定词/范围被忽略（"两份均""所有""具体"等全称量词）
- **fc_a_001/004/007/012 选项 D、fc_a_016 选项 A**：选项要求"两份文档**均**提到 X"或
  "其中一份记载了**具体数值** Y"，judge 只在**一份**文档找到 X 就判 supported，
  没校验全称量词覆盖到所有 gold 文档。
- **reg_a_019 选项 D、reg_a_009 选项 A**：法规细节（具体日期/数值/主体）judge 凭一条
  相近法条就 supported，没核对该法条是否真对应选项所述的那部 办法/那条款。

### 套路 C：数值方向/单位错位
- 多见于 mcq（见根因 2）。多选里 res 系列把"市占率全球第一 vs 中国大陆第一"、
  "微降 8.47% vs 增长 8.47%"判反的，judge 大多 refuted 对了，少数没抓住。

### 修复方向（不针对单题，针对这类）
1. **judge 增加"主语一致性"硬校验**：判 supported 前，必须确认
   `证据所属 doc_id == 选项点名的文档/主体`，且证据陈述的主体与选项主体一致。
   跨文档题尤其要把"选项点名 fc_text_007 / 韩国 / 芯原"与证据 doc/主体做匹配门控。
2. **全称量词检测**：选项含"均/都/所有/两份…均/全部"时，要求**每个 gold 文档**都有
   独立支撑证据，否则不得判 supported（缺一即 refuted/insufficient）。
3. **禁止 reason 与 status 矛盾**：已有 `_reason_contradicts_supported`，
   但 fc_a_019 漏网——应强化：reason 里出现"并未出现/未找到/属另一文档/存疑"等
   否定线索时，强制 status 不得为 supported（降级或触发复判）。
4. **多选默认收紧**：对置信不满或证据非"逐字命中选项关键事实"的 supported，
   多选题应倾向**不选**（过包含比欠包含在本批更致命：13 vs 7）。

## 四、根因 2：mcq/tf 数值与逻辑判定（6 题）

- **fin_a_008 / fin_a_015**：财报数值口径错配。如"每 10 股派 45.53 元"被选项写成"每股 45.53 元"，
  或"现金分红占比 20%（年度）vs 50%（年度+特别）"。judge 大多 refuted 对了，
  但最终 mcq 仍选了 refuted 项（fin_a_008 pred=D，D 是 refuted）——
  **mcq 全 refuted 时的兜底选择逻辑有问题**：应回退到"反驳证据最弱/最可能为真"的那项，
  而非随机/默认。fin_a_015 四个选项全 refuted，pipeline 仍输出 B。
- **ins_a_006**：保险赔付**多步数值计算**（共享免赔额、补偿原则）。judge 把 C 判 supported、
  A 判 refuted，但 gold=A。这是**算术推理**错误，非检索问题。需要数值校验器介入
  （太保是否赔 0.2 万的补偿原则计算）。
- **fc_a_013 / fc_a_018 / res_a_018**：tf 判断题。pipeline 其实判对了 2 个（013/018 的 B、018 的 A），
  被人工改错（见第二节）。fc_a_018 两选项全 insufficient——tf 题缺一份文档证据时
  **不应判 insufficient 后乱猜**，应有明确兜底（"无法证实'晚于'→陈述不成立→选'错误'"）。

### 修复方向
1. **mcq/tf 兜底**：全 refuted / 全 insufficient 时，不要落到默认字母，
   按"反驳强度最低"或"语义最接近证据"的选项决策，并记录 stop_reason。
2. **财报/保险数值题接数值校验器**：单位（元/万元/亿元、每股/每10股）、
   占比口径（年度 vs 累计）、方向（增/降）做结构化抽取后比对，别让 LLM 自由判。

## 五、根因 3：多选欠包含（7 题）+ 跨文档召回失衡

- **ins_a_008/012/019、fin_a_020、res_a_001、ins_a_014、fc_a_014**：漏选项多被判
  `insufficient` 或 `refuted`。其中 ins_a_019 选项 C、ins_a_012 选项 B、fin_a_020 选项 C
  是"证据没检索到对应条款"→ judge insufficient → 漏选。
- 这与之前发现的 **STARVED/WEAK 跨文档召回失衡**部分重叠（res 系列 pack2_text01 常只 2-4 卡）。
  **已做过失败实验并回退**：在 `evidence_selector.select()` 里对命中不足的 gold 文档做全局补召回，
  单测 STARVED 4→0、170 单测全过，但端到端净负面（fc_a_011 AB→A、ins_a_015 AB→ABD 倒退，
  仅 ins_a_008 AC→ABC 受益，net -1）。原因：补进的是"该文档的任意证据"而非"支撑特定选项的那条",
  在多选判定里引入噪声。当前 `select()` 已回退该改动，仅留一条注释记录。
- 真正问题：漏选项的证据**确实存在于 gold 文档**，但要么没检索到那一条（召回粒度），
  要么检索到了但 judge 因"条款表述不够直白"判 insufficient（判定保守）。

### 修复方向
- 对判 `insufficient` 的选项，做**选项锚定的定向补检索**（只在该选项点名的 gold 文档内，
  用选项关键词二次检索），再复判一次。注意：补进来的证据**只喂给该选项的复判**，
  不混入全局证据池，避免重蹈"全局补召回引噪声"的覆辙。
- 这正是上一轮失败实验的修正版：**定向、隔离、只复判单个 insufficient 选项**，
  而非全局回填。需端到端验证而非只看召回指标。

## 六、优先级建议（按杠杆排序）

1. **【最高】根因 1 套路 A+B**：主语一致性门控 + 全称量词检测 + reason/status 矛盾强校验。
   覆盖 13 题过包含里的大多数，且是"这类题"的通用解。
2. **【高】禁止无据人工加选项**：流程纪律，零成本，直接消除 5 个人工倒退。
3. **【中】mcq/tf 兜底逻辑** + 财报/保险数值校验器：覆盖 6 题单选/判断。
4. **【中】insufficient 选项的隔离式定向补检索 + 单选项复判**：覆盖 7 题欠包含，
   必须端到端验证（吸取上轮教训）。

**financial_reports 召回 100%、判定相对稳，不要动检索层；问题在判定层。**

## 七、证据归属准确度 —— 具体落地设计（根因 1 的工程化）

现有基础设施（`agent/answer_normalizer.py`）：
- `judgements[L].evidence_ids` → 可映射到 `evidence_cards[*].doc_id`（已验证，如 res_a_004 的 C → E9）；
- `_reason_contradicts_supported` / `downgrade_self_refuting_judgements`：基于中文短语黑名单，**脆弱**，
  fc_a_019 漏网（reason 写了"text07 中并未出现安克创新"却仍 supported）；
- `domain_answer_review`：已有 `over_select` / `unsupported_answer_letter` 等告警 code，但只告警不阻断。

把"归属准确度"拆成**三层闸门**，按可机械化程度从强到弱：

### 闸门 1（强、可机械化）：doc 级归属门控
选项显式点名某文档时（正则提取 `fc_text_\d+` / `第[一二]份文档` / 主体名如"芯原/宁德时代/中国移动"），
判 supported 前要求其 `evidence_ids` 至少一条来自**该文档对应的 doc_id**。
不满足 → 降级 insufficient 并触发该选项定向复判。
- 直接命中 fc_a_019（D 的证据全来自 text04，但选项点名第二份 text07）。
- 落点：`a_leaderboard_agent._judge` 拿到 judgements 后、`synthesize_answer` 前，
  新增 `enforce_doc_attribution(judgements, options, evidence_cards, doc_ids)`。

### 闸门 2（中、可机械化）：全称量词覆盖检测
选项含"均/都/两份…均/所有/全部"时，要求**每个 gold doc** 都有 supported 该选项的独立证据；
缺任一 → 不得 supported。
- 命中 fc_a_001/004/007/012 的 D、fc_a_016 的 A（"两份均提到具体违约/某指标"）。
- 落点：同上，新增 `enforce_universal_quantifier(...)`，量词词表 + 逐 doc 证据计数。

### 闸门 3（弱、需 prompt 纪律）：主语一致性
res_a_002/004/012 是"数值对、主语错"（博通预测被说成芯原、台湾增速被说成韩国/全球）。
doc 级闸门拦不住（同一份研报里既有博通也有芯原）。只能靠 prompt 强约束：
- 判 supported 前必须在 reason 里**逐字回填证据中的主语**，并与选项主语串比对；
- 输出字段增加 `subject_in_evidence` / `subject_in_option`，不一致则 judge 自身判 refuted。
- 这层无法纯后处理兜底，需改 judge prompt + 增一个轻量后校验（两主语字符串不重叠则告警复判）。

### 验证铁律（吸取上轮教训）
任何闸门改动**必须端到端复跑全 100 题**对比 gold，单测/中间指标通过不算数。
上次"STARVED 4→0、170 单测全过"但端到端 net -1 的教训不能重演。
优先用本批 13 道过包含题 + 已知正确题做回归集，确认**只减过包含、不误伤正确 supported**。
